# 11 - 需求与实现对照（维护说明）

> **本文作用**：在通读 `docs/01`～`10` 与浏览 `web/src` 后，用**代码事实**记录「需求写了什么、当前实现到哪、差在哪里」。  
> **权威关系**：**同一订单内「支付组」收款边界与统计口径**以 **`docs/CONSTITUTION_支付组.md`** 为准（与 `docs/04` 订单管理章节互为补充：`04` 写界面与落地、`CONSTITUTION` 写不可违背的规则）。**数据字段**仍以 **`docs/06` 数据模型** 与 **`docs/08` 第一版范围（产品目标）** 为准；`docs/08` 里大量 `[x]` 表示**产品规划上第一版要覆盖的能力**，**不等于**本仓库每一行都已实现。  
> **更新时机**：里程碑、大模块合入、或发现本文与代码明显不符时，请改本文并 `git commit`（`docs:`）。

## 二、路由 → 页面 → 数据与说明

以下均指 `web/src/appRoutes.tsx` 中声明的路由与对应页面文件。

| 路径（模式） | 页面组件 | 主要数据源 | 备注 |
|--------------|----------|------------|------|
| `/` | `pages/Home.tsx` | 无（占位链接） | 文案仍为「占位」，非营销首页 |
| `/login` | `pages/Login.tsx` | Firebase **匿名登录** | **非** docs/05 完整手机号/邮箱流程 |
| `/register` | `pages/Register.tsx` | 无 | **占位**，未接验证码 |
| `/invite/:code` | `pages/InviteAccept.tsx` | Firestore 邀请 + 权限 | 与 `invitationService` 等配合 |
| `/dashboard` | `pages/merchant/ShopList.tsx` | `shops` | 店铺列表 |
| `/dashboard/:shopSlug` | `pages/merchant/Dashboard.tsx` | `getShopBySlug` + `listOrdersByShopId` + `aggregateShopOrdersForToday` | 今日三格（已确认/待核实/未付款）按**本日创建**订单筛 |
| `/dashboard/.../projects` | `pages/merchant/ProjectList.tsx` | `projects` | 新建草稿、进入编辑 |
| `/dashboard/.../projects/new` 等 | `pages/merchant/ProjectEdit.tsx` | `projects`、配送点库等 | 发布/截单等以 `projectService` 为准 |
| `/dashboard/.../orders` | `pages/merchant/OrderManagement.tsx` | `listOrdersByShopId` | Tab 为 **全部/进行中/已完成/已取消**（与 docs/04 四种状态命名不必字面对齐） |
| `/dashboard/.../order/:projectId/:orderNumber` | `pages/merchant/MerchantOrderDetail.tsx` | `getOrderByNumber` + 确认收款等 | 有 **确认收款**、内部备注、凭证展示；**无**「一键拒绝」独立动作（见下节） |
| `/dashboard/.../reconciliation` | `pages/merchant/ReconciliationStatement.tsx` | 订单 + `reconciliationSummary` | 复制、**CSV** 下载；**非**必须已接 xlsx 库 |
| `/dashboard/.../delivery-points` | `pages/merchant/DeliveryPoints.tsx` | 配送点 | 以代码为准 |
| `/dashboard/.../admins` | `pages/merchant/AdminManagement.tsx` | 邀请 + 权限 | 三级管理员与邀请码流程 |
| `/dashboard/.../settings` | `pages/merchant/ShopSettings.tsx` | 无 | **占位**（店名/主题色/付款码等见 docs/04，**未在 UI 实现**） |
| `/shop/:shopSlug/:projectId` | `pages/customer/ShopHome.tsx` | **默认** `loadShopHomeFromFirestore`；`?mock=1` 为 `mockShopHome` | 与 `shopHomeService` 一致 |
| `/shop/.../order` | `pages/customer/OrderForm.tsx` | `createOrder` → `orders` + 事务扣库存 | — |
| `/shop/.../my-orders` | `pages/customer/MyOrders.tsx` | `listOrdersByCustomer` | 状态与截图提示已做 |
| `/shop/.../orders/:orderId/add-items` | `pages/customer/OrderAppend.tsx` | `getOrderByNumber` + `appendOrderItems` | 顾客加菜入口已存在（是否完全满足 docs/08 仍以流程细测为准） |
| `/shop/.../orders/:orderId` | `pages/customer/OrderDetail.tsx` | `getOrderByNumber` | 截图上传/删除、状态展示 |
| `/dashboard/.../cards`、`/dashboard/.../cards/:templateId` | `pages/merchant/CardTemplates.tsx`、`CardTemplateDetail.tsx` | `cardService` + `card_templates` | 商户侧次卡/储值卡模板管理 |
| `/shop/:shopSlug/cards`、`/shop/:shopSlug/cards/buy/:templateId`、`/shop/:shopSlug/cards/topup/:cardId` | `pages/customer/CustomerCards.tsx`、`CustomerCardBuy.tsx` | `cardService` + `customer_cards` + `card_purchase_requests` | 顾客侧卡列表、购卡、充值流程 |

**说明**：顾客端身份为 **`customerIdentity`（localStorage customerKey）**，非账号体系登录。

## 四、与「第三节」相关的建议优先级（供 HANDOVER 第三节迭代）

1. **对齐产品**：决定 docs/08 中哪些仍为「目标」、哪些改为「第二版」，避免 checklist 与仓库长期打架。  
2. **上线必需**：Firestore/Storage 规则、真实链路验收（见 `08` 上线判断标准）。  
3. **高 visible 缺口**：`ShopSettings`、`Register/Login` 与 docs/05、分享卡 / PWA（若仍属 v1）。  
4. **体验增强**：`Home` 非占位、PWA、实时监听（按需）。
