## 三、当前产品形态（比初版文档多出来的部分）

除「商户自营店铺 + 顾客 `/shop/...`」主链外，仓库已包含：

1. **大马饭团（顾客侧）**  
   - 路径前缀 **`/feituan`**：主页、项目页、**购物车 / 合并结算 / 合并付款**、**消息**、钱包与充值、我的饭团订单等（路由见 `appRoutes.tsx`）。  
   - 部分下单与订单页**复用**自营顾客组件（如 `OrderForm`、`OrderDetail`），挂在 `/feituan/projects/:projectId/...` 下；饭团 **`FeituanMyOrders`** 为独立列表页。  
   - **主页**：项目卡片图块为自动马赛克排版（`feituanHomeMosaic.ts`）。  
   - **消息 Tab**：角标 = 待付款（含尾款）+ 未读通知；离开消息页后未读清零；列表待付款置顶、橙色标记（`feituanMessages.ts`）。  
   - **长期项目**：合并付款页可改配送档；我的订单卡片展示配送时间；订单详情支持 `returnTo` 返回来源页。  
   - 细节与文件表见 **`HANDOVER.md` §八**、**`docs/11` §二（饭团顾客）**。

2. **饭团 / 平台管理后台**  
   - 路径前缀 **`/admin/feituan/...`**：饭团管理、订单列表、钱包后台、配送、成本、对账、项目只读预览、饭团订单详情（复用 `MerchantOrderDetail` 的饭团模式）等。  
   - **`/admin/registrations`、`/admin/shops`**：平台侧登记与店铺相关页面。

3. **微信服务号相关能力**  
   - Hosting 将 `/wechat`、`/wechat/oauth/*`、`/api/wechat/*` 等重写至 **Cloud Functions**（见根目录 **`firebase.json`**）。  
   - 前端辅助逻辑见 **`web/src/lib/wechatService.ts`**、hooks 如 `useWechatNotifySession` 等；订单文档上已有与通知会话相关的字段（见 `OrderDoc` 类型）。

4. **支付与商业扩展（店铺侧 + 订单侧）**  
   - **店铺次卡 / 储值卡**模板与顾客购卡、充值流程（商户 `dashboard/.../cards`，顾客 `/shop/.../cards`）。  
   - **饭团钱包**（充值档位、凭证、流水、后台审核）。  
   - **产品库**、**套餐（bundleTools）** 等在类型与编辑流中已扩展；与初版「仅单品」文档有差异。

5. **站点根路径 `/` 行为**  
   - 现为 **`MerchantEntry`**：未登录说明页；已登录用户会跳转 **`/dashboard`**。与早期 README 中「平台营销首页」示意图**不一致**，以路由为准。

## 四、路由总览（按前缀，便于搜代码）

**权威列表仍以 `web/src/appRoutes.tsx` 为准**；下表为分组速查。

| 前缀 | 主要用途 |
|------|----------|
| `/` `/home` `/login` `/register` `/invite/*` `/account` | 入口、登录注册、邀请、账号中心 |
| `/dashboard` `/dashboard/:shopSlug/...` | 商户后台：看板、项目、订单、对账、配送点、管理员、设置占位、次卡、产品库等 |
| `/shop/:shopSlug/...` | 顾客自营店：项目首页、下单、订单、加菜、购卡/充值 |
| `/feituan/...` | 饭团顾客侧：主页、项目、钱包、我的订单等 |
| `/admin/feituan/...` | 饭团管理：订单、对账、配送、成本、预览等 |
| `/admin/registrations` `/admin/shops` | 平台侧页面 |
| `*` | `NotFound` |
