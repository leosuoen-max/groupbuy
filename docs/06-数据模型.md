# 06 - 数据模型

> 基于 Firebase Firestore 设计。Firestore 是 NoSQL 文档型数据库，集合（Collection）下是文档（Document）。

---

## 6.1 总体架构

```
firestore/
├── users/                       注册用户
├── shops/                       店铺
├── projects/                    项目（菜单/链接）
├── orders/                      订单
├── permissions/                 权限关系（用户-项目-角色）
├── delivery_points/             配送点（按店铺分组）
├── invitations/                 管理员邀请链接
├── feedback/                    用户反馈
└── system/                      系统配置
```

---

## 6.2 集合详细设计

### 6.2.1 users（用户）

```typescript
{
  id: string,                    // 系统生成
  phone: string,                 // 手机号 +60123456789（含国际区号）
  email?: string,                // 邮箱（选填，用于 Magic Link）
  nickname?: string,             // 昵称（默认 "用户***"）
  avatar?: string,               // 头像 URL
  createdAt: Timestamp,          // 注册时间
  lastLoginAt: Timestamp,        // 最后登录时间
  
  // 来自微信内访问的额外信息
  wxOpenId?: string,             
  wxNickname?: string,
  wxAvatar?: string,
}
```

### 6.2.2 shops（店铺）

```typescript
{
  id: string,                    // 系统生成
  slug: string,                  // URL 标识（unique），如 "huijie"
  name: string,                  // 店名 "辉姐家常小厨"
  ownerId: string,               // 创建人 userId
  
  // 视觉设置
  logo?: string,                 // Logo URL
  bannerImage?: string,          // 门头图片 URL
  themeColor: string,            // 主题色，8 选 1
  
  // 营业信息
  description?: string,          // 营业说明
  paymentMethods: [              // 付款方式
    {
      id: string,
      name: string,              // "TNG"、"DuitNow" 等
      qrCodeUrl: string,         // 收款码图片 URL
    }
  ],
  
  // 设置
  settings: {
    language: 'zh' | 'en' | 'ms',
    currency: 'MYR' | 'CNY' | ...,

  // 元数据
  createdAt: Timestamp,
  updatedAt: Timestamp,
  isActive: boolean,             // 是否启用
}
```

### 6.2.3 projects（项目）

```typescript
      url: string,

  // 商品列表
  products: [
    {
      id: string,
      name: string,              // 商品名称
      description?: string,      // 一行说明

      discountPrice?: number,    // 活动价

      discountEnd?: Timestamp,   // 活动止
```

### 6.2.4 orders（订单）

```typescript
{
  id: string,
  projectId: string,             // 所属项目
  shopId: string,                // 所属店铺（冗余字段，方便查询）

  // 顾客信息
  customerId?: string,           // 注册用户 ID（游客为 null）
  customerName: string,
  customerPhone: string,
  customerAddress: string,
  customerNote?: string,
  
  // 商品明细
  items: [
    {
      productId: string,

      quantity: number,
```

### 6.2.5 permissions（权限）

记录"用户在某个项目里的角色"。

```typescript
{
  id: string,
  userId: string,
  projectId: string,             // 也可以是 shopId（针对店铺级别权限）
  scope: 'shop' | 'project',     // 作用域
  scopeId: string,               // shopId 或 projectId
  role: 'owner' | 'high_admin' | 'normal_admin',
  grantedBy: string,             // 邀请人 userId
  grantedAt: Timestamp,
  invitationId?: string,         // 来自哪个邀请
}
```

**注意：** 
- 创建人在 `shops` 表的 `ownerId` 字段已经有记录，不需要额外在 permissions 里再存
- permissions 表存的是被邀请的管理员
- 顾客身份不存权限表，通过 orders 表的 customerId 自然识别

## 6.3 索引设计

需要为以下查询建立 Firestore 复合索引：

```
orders:
- (projectId, status, createdAt desc)
- (projectId, deliveryPointId)
- (customerId, createdAt desc)
- (shopId, createdAt desc)

projects:
- (shopId, status, createdAt desc)

permissions:
- (userId, scope, scopeId)
- (scopeId, role)
```

## 6.4 关键查询场景

### 6.4.1 顾客打开项目链接

```typescript
1. 通过 slug 查 shop:
   db.collection('shops').where('slug', '==', 'huijie').get()

2. 通过 projectId 查 project:
   db.collection('projects').doc(projectId).get()

3. 查询当前用户身份（如果已登录）:
   db.collection('permissions')
     .where('userId', '==', currentUserId)
     .where('scopeId', '==', projectId)
     .get()

4. 实时监听项目状态（截止状态、报名人数等）:
   db.collection('projects').doc(projectId)
     .onSnapshot(...)
```

### 6.4.2 商户查看待核实订单

```typescript
db.collection('orders')
  .where('projectId', '==', projectId)
  .where('status', '==', 'pending')
  .orderBy('createdAt', 'asc')
  .get()
```

### 6.4.3 顾客查看自己的订单

**已注册用户：**
```typescript
db.collection('orders')
  .where('customerId', '==', userId)
  .orderBy('createdAt', 'desc')
  .get()
```

**游客（通过浏览器 Cookie）：**
```typescript
// localStorage 存储 orderIds 数组
db.collection('orders')
  .where('id', 'in', localStorageOrderIds)
  .get()
```

## 6.5 数据保留策略

### 第一版

**全部永久保留**，简单粗暴：
- 订单数据
- 付款截图
- 商户上传的图片
- 草稿

### 第二版（用户量起来后）

- 草稿箱 60 天未编辑提醒
- 1 年前的图片自动压缩到低画质
- 注销账户的所有数据立即删除

### 合规要求（马来西亚 PDPA）

- 提供"删除我的账户"功能
- 用户协议中说明保留策略
- 删除请求 30 天内执行

## 6.7 安全规则（Firestore Security Rules）

关键原则：
- 顾客只能读自己创建的订单
- 顾客可以创建订单（无需注册）
- 商户/管理员通过 permissions 表验证身份
- 付款截图只有订单关联用户和店铺管理员可见

伪代码示例：

```
match /orders/{orderId} {
  allow read: if 
    // 订单创建者
    request.auth.uid == resource.data.customerId
    // 或者是该项目的管理员
    || isProjectAdmin(request.auth.uid, resource.data.projectId)
    // 或者游客通过 localStorage 验证（前端处理）
    || resource.data.customerId == null;
    
  allow create: if true;  // 任何人可创建订单
  
  allow update: if
    // 顾客只能更新自己的订单的特定字段
    (request.auth.uid == resource.data.customerId 
     && onlyChangesAllowedFields(request))
    // 管理员可更新状态
    || isProjectAdmin(request.auth.uid, resource.data.projectId);
}
```
