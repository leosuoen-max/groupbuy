# 01 - 产品概述

## 产品定位

为微信群、WhatsApp 群等社交群组的团购/外卖商户设计的 H5 订单管理系统。

- **顾客**：通过链接直接访问，无需安装 App
- **商户**：通过 H5+PWA 后台管理订单、菜单、配送点

支持微信、WhatsApp、Telegram、Line 等所有社交平台。

---

## 真实使用场景

### 场景 1：辉姐家常小厨（核心场景）

**主角：** 辉姐，在马来西亚吉隆坡 Kepong 区做东北家常菜外卖

**每日工作流：**

```
12:00 PM
└─ 在家用 PWA 打开商户后台
   ├─ 复制昨天的"晚餐菜单"项目
   ├─ 修改日期、调整菜品
   ├─ 添加 1-2 道特色菜
   └─ 点"发布"

12:05 PM  
└─ 生成分享卡片
   └─ 保存到相册 → 发到 5 个微信群 + 3 个 WhatsApp 群

12:30 PM - 5:00 PM（接单期）
└─ 顾客陆续下单、上传付款截图
   └─ 辉姐时不时打开后台
      ├─ 看新订单
      ├─ 核对付款截图（三色提示帮助判断）
      └─ 处理特殊请求（如修改备注）

5:00 PM
└─ 自动截单（或辉姐手动截单）
   └─ 后台显示出餐清单
      ├─ 按配送点分组
      ├─ 配送点 1（A座大堂）：5 单
      ├─ 配送点 2（B座入口）：8 单
      └─ 配送点 3（自取）：3 单

5:30 PM
└─ 开始做饭

6:00 PM - 7:00 PM
└─ 配送
   └─ 截图出餐清单发给送餐员

7:30 PM
└─ 一天结束
   └─ 打开对账单
      ├─ 已确认到账：RM 215.4
      ├─ 待核实：RM 38.4
      └─ 未付款：RM 19.6（明天联系顾客）
```

### 场景 2：临时团购

**主角：** 普通用户小明，想组织一次办公室团购

**使用流程：**

```
1. 注册账号（免费）
2. 创建一个"临时项目"
   └─ 比如"周五烧烤聚餐报名"
3. 设置选项、价格
4. 发到办公室群
5. 同事们点链接报名
6. 截止后查看名单和金额
7. 收完钱，项目自动归档
```

## 核心价值

### 对商户

| 痛点 | 我们的解决方案 |
|---|---|
| 微信群消息刷得太快，订单容易漏 | 顾客直接在链接里下单，不靠聊天记录 |
| 收款核对要翻聊天找截图 | 截图集中显示，三色标记重点 |
| 配送地址五花八门，要手动归类 | 配送点系统，顾客自己选 |
| 想搞促销手动改价麻烦 | 早鸟活动价，到时间自动恢复 |
| 每天发新菜单要重新填一遍 | 一键复制昨天的菜单 |
| 和员工分工管理麻烦 | 三级管理员系统 |

### 对顾客

| 痛点 | 我们的解决方案 |
|---|---|
| 在群里要回复很多次才下完单 | 链接里一次搞定 |
| 不知道自己点了什么 | "我的订单"清晰显示 |
| 老板没看到我付款 | 主动上传截图，看到状态 |
| 改主意要重新刷一长串消息 | 直接在订单里加菜/改备注 |

### 对平台

- 不抢商户流量（不显示平台 Logo）
- 不污染顾客体验（不塞广告）
- 商户用得越顺，越愿意推荐给同行
- 用户基数起来后再考虑商业模式

---

## 商业模式

### 第一阶段（当前）

**完全免费**，目标是积累商户用户，验证产品价值。

### 第二阶段（用户量起来后）

可能的方向（先不做决定）：
- 增值服务（如更高级的数据分析、更多分享卡片样式）
- 推广分成（帮商户引流，按订单抽成）
- 商家工具市场（如发票打印、配送对接等）

**坚持不做：**
- 在顾客端塞广告
- 卖商户/顾客数据
- 强制收年费

## 目标用户画像

### 商户

| 维度 | 描述 |
|---|---|
| 规模 | 个人或夫妻档，1-3 人经营 |
| 业务 | 家庭外卖、社区团购、办公室代购、班级活动 |
| 地点 | 起点：马来西亚（华人聚集区）|
| 已有方式 | 群报数小程序、Excel、手工记账 |
| 痛点 | 订单管理乱、收款核对慢、想跨平台用 |

### 顾客

| 维度 | 描述 |
|---|---|
| 关系 | 是商户的邻居、同事、朋友、熟客 |
| 习惯 | 用微信群、WhatsApp 群和商户互动 |
| 期望 | 下单方便、不重复填资料、能随时查看 |
