# 02 - 用户角色与权限

## 核心原则

**单一身份模型：一个手机号 = 一个账号 = 多种能力**

不分"顾客账号""商户账号"，注册了就拥有所有能力，用不用是用户自己的事。

**身份是绑定到具体项目（链接）的，不是绑定到人的。**

---

## 五种角色

### 1. 普通游客（未注册）

**定义：** 没有注册账号的访客，或者注册了但跟当前项目没关系的人

**能做的：**
- 浏览商户的项目页面
- 选菜下单
- 填写姓名、地址、电话、备注
- 上传付款截图
- 修改自己的订单（加菜、改备注、改地址、改配送点）
- 取消自己的订单（待付款前）
- 提交意见反馈

**不能做的：**
- 跨设备查看历史订单（清缓存就丢失订单引用）
- 跨项目查看自己在其他店铺的订单

**身份识别：**
- 浏览器 Cookie + LocalStorage
- 微信内打开时尝试获取昵称头像（如果允许）

---

### 2. 已注册顾客（注册了但只下过单）

**定义：** 注册了账号，在某些项目里下过单

**比游客多的能力：**
- 跨设备查看自己的历史订单（任何设备登录都能看）
- 跨项目查看（在 A 店铺、B 店铺下过的单都能查）
- 订单状态变化时收到通知

**身份识别：**
- 通过手机号登录后系统识别
- 系统在权限表中记录"该用户在哪些项目里下过单"

---

### 3. 普通管理员（被邀请加入某个项目）

**定义：** 被项目创建人邀请，协助处理订单

**能做的（限定在这个项目内）：**
- 普通顾客的所有功能
- 查看所有订单
- 确认/拒绝付款
- 添加管理员内部备注（仅管理员可见）
- 导出对账单
- 手动匹配配送点

**不能做的：**
- 编辑项目内容（菜单、说明、商品）
- 修改店铺设置
- 添加/删除其他管理员
- 截单/重新开放（待讨论是否开放）

**身份识别：**
- 注册账号 + 在权限表中记录角色

---

### 4. 高级管理员（被邀请加入某个项目）

**定义：** 类似项目的合伙人，与创建人同等权限（除了几个限制）

**能做的（限定在这个项目内）：**
- 普通管理员的所有功能
- 编辑项目内容（菜单、商品、说明）
- 修改项目设置
- 手动截单/重新开放
- 修改店铺设置（店名、Logo、配送点等）

**不能做的：**
- 添加/删除其他管理员（仅创建人）
- 删除项目/店铺（仅创建人）
- 转让创建人身份（仅创建人，第二版功能）

---

### 5. 创建人（项目所有者）

**定义：** 创建项目/店铺的人，拥有最高权限

**能做的（在自己的项目内）：**
- 高级管理员的所有功能
- 添加/删除任何级别的管理员
- 删除项目/店铺
- 转让创建人身份（第二版功能）

**特殊规则：**
- 一个项目只有一个创建人
- 创建人可以转让给其他注册用户
- 转让后原创建人变成高级管理员

## 身份识别系统

### 如何判断访问者身份

每次有人打开 `yourapp.com/shop/[shop]/[project]`：

```
1. 检查浏览器登录状态
   ├─ 未登录 → 游客视角
   └─ 已登录 → 继续判断

2. 已登录用户 → 查询权限表
   "用户ID + 项目ID" 在表里是什么角色？
   ├─ 创建人 → 显示创建人视角
   ├─ 高级管理员 → 显示高级管理员视角
   ├─ 普通管理员 → 显示普通管理员视角
   ├─ 顾客（下过单）→ 显示顾客视角
   └─ 没记录 → 显示游客视角（实际是已注册的普通游客）
```

### "更多"菜单的差异化显示

不同身份打开同一个项目链接，"更多"菜单内容不同：

**游客 / 已注册顾客（未下过单）：**

## 同一用户在不同项目的角色

举例：注册用户"小明"

```
小明（一个手机号、一个账号）
├── 在「辉姐家常小厨 · 5/4 晚餐」 = 创建人
├── 在「张记包子铺 · 周末团购」 = 高级管理员（被张老板邀请）
├── 在「李家烘焙坊 · 蛋糕预订」 = 普通管理员（被邀请帮忙）
├── 在「王哥水果铺 · 5/3 水果」 = 顾客（下过 1 单）
└── 在其他陌生项目 = 普通游客（如果点进去）
```

**关键：** 系统通过"用户ID + 项目ID"组合查询角色，每个组合独立。

## 管理员邀请流程

### 第一版：邀请链接方式

```
1. 创建人在管理员管理页面点"添加管理员"
2. 选择权限级别（普通/高级）
3. 系统生成专属邀请链接（24小时有效）
   例：yourapp.com/invite/xxxxxx
4. 创建人复制链接，发给对方（微信/WhatsApp）
5. 对方点开链接：
   ├─ 已注册 → "您被邀请成为 XX 的高级管理员，是否接受？"
   └─ 未注册 → 引导注册（手机号验证码），注册成功自动接受
6. 接受后立即生效，写入权限表
```

### 第二版：手机号邀请

直接输入对方手机号，系统给对方推送邀请通知。

## 删除/调整管理员

```
创建人在管理员列表中：
├── 点击某个管理员旁的 [⋯] 按钮
├── 弹出选项：
│   ├── 设置为普通管理员
│   ├── 设置为高级管理员
│   ├── 删除该管理员
│   └── 转让创建人（第二版）
└── 操作后立即生效
```

被删除的管理员：
- 立即失去对该项目的管理权限
- 之前以管理员身份做过的操作（如确认订单）保留记录
- 如果他在该项目下过单，订单数据保留，他变回顾客身份
