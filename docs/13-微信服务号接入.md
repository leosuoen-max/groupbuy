# 微信服务号接入说明

## 当前阶段目标

先完成服务号网页授权基础接入：

- 微信网页 OAuth 获取 `openid`
- 已手机号登录用户绑定服务号 `openid`
- 微信内访问饭团/订单页面时，自动获取订单通知会话
- 为后续订单消息、配送消息打基础

手机号仍是钱包、商户和管理员权限的正式身份；微信 `openid` 只作为消息通知通道。

## 需要配置的环境变量

Cloud Functions 当前需要以下环境变量：

```bash
WECHAT_APP_ID=服务号 AppID
WECHAT_APP_SECRET=服务号 AppSecret
WECHAT_ORDER_SUBMITTED_TEMPLATE_ID=订单提交通知模板 ID
SHARE_APP_ORIGIN=https://你的线上域名
```

当前默认线上域名是：

```text
https://groupbuy-app-24c46.web.app
```

## 网页授权域名

在「设置与开发」→「公众号设置」→「功能设置」里配置：

```text
groupbuy-app-24c46.web.app
```

如果后续使用自定义域名，就改成自定义域名，不要带 `https://` 和路径。

微信会要求下载 `MP_verify_*.txt` 校验文件，并上传到 H5 根目录。项目里应放到：

```text
web/public/MP_verify_xxx.txt
```

部署 Hosting 后，确保能访问：

```text
https://groupbuy-app-24c46.web.app/MP_verify_xxx.txt
```

## 可选：服务器配置

如果后续需要接收关注/取关、扫码、用户发消息等微信事件，再配置服务器 URL。

可选环境变量：

```bash
WECHAT_TOKEN=你在服务号服务器配置里填写的 Token
```

服务号后台「设置与开发」→「基本配置」：

```text
服务器 URL：https://groupbuy-app-24c46.web.app/wechat
Token：和 WECHAT_TOKEN 一致
消息加解密方式：明文模式
```

## 当前已接入的路径

- `/wechat`：服务号服务器校验入口
- `/wechat/oauth/start`：发起网页 OAuth
- `/wechat/oauth/callback`：接收微信 OAuth code
- `/api/wechat/bind`：登录用户确认绑定 openid

OAuth 有两种模式：

- `mode=account_bind`：账号中心显式绑定手机号账号与 `openid`
- `mode=session`：微信内访问 H5 时静默拿到通知会话，订单会保存 `wechatNotifyOAuthStateId`

前端入口在：

- `/account`
- `/feituan/account`
- `/feituan/my-orders`

饭团首页、饭团项目页、订单列表和订单详情会在微信内自动尝试静默授权。下单页不会主动跳转授权，避免丢失购物车页面状态；它只会读取前面页面已经保存的微信通知会话。

## 当前已发送的模板消息

第一版只发送一条：

> 您提交了新订单，请关注状态更新。

触发时机：饭团订单创建成功后。

需要在服务号后台申请/选择一个模板，并把模板 ID 配到：

```bash
WECHAT_ORDER_SUBMITTED_TEMPLATE_ID=模板ID
```

当前代码按以下字段发送：

- `first`：您提交了新订单，请关注状态更新。
- `keyword1`：项目名 + 订单号
- `keyword2`：项目名称
- `keyword3`：订单金额
- `remark`：点击查看订单详情。

如果选用的模板字段不同，微信会返回发送失败；不影响用户下单，但订单上会记录发送失败原因。
