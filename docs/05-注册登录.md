# 05 - 注册与登录

## 核心原则

**注册不分类别。** 一个手机号 = 一个账号 = 多种能力。

注册后：
- 可以管理自己的订单（如果下过单）
- 可以创建自己的店铺、发布项目
- 可以接受管理员邀请

不强制顾客注册。游客也能下单。

### 5.4.2 备选方式：邮箱 Magic Link

```
┌─────────────────────────────────────┐
│ 用邮箱登录                          │
│                                     │
│ 邮箱：                              │
│ [_______________________]           │
│                                     │
│ [ 发送登录链接 ]                     │
│                                     │
│ 我们会发一个登录链接到你的邮箱        │
│ 点击链接即可登录，无需密码            │
│                                     │
│ ─────────────                       │
│ 用手机号登录 →                       │
└─────────────────────────────────────┘
```

**Magic Link 流程：**
1. 用户输入邮箱
2. 系统发邮件，含一个登录链接
3. 用户在任意设备打开邮件，点击链接
4. 链接 10 分钟内有效
5. 点击后跳转到 yourapp.com，自动登录
6. 链接用完即失效

**安全特性：**
- 链接只能用一次
- 10 分钟有效期
- 与手机号绑定（一个账号既能用手机也能用邮箱）

## 5.5 登录体验优化

### 5.5.1 30 天长期登录

- 默认勾选"30 天内自动登录"
- 浏览器存储一个长期 token
- 30 天内打开 yourapp.com 自动进入后台
- 商户每天打开后台基本无感登录

### 5.5.2 多设备同步

- 一个账号可在多个设备同时登录
- 数据实时同步（基于 Firestore）
- 任意设备登出不影响其他设备

### 5.5.3 免登录预览

- 商户即使没登录，也能"以顾客视角"看自己的店铺
- 因为店铺链接本来就是公开的
- 需要操作时（编辑、查订单）才提示登录

## 5.6 找回账号

### 5.6.1 忘记登录方式

无密码方案 → 没什么需要"找回"的

```
忘记账号？
- 用注册时的手机号 + 验证码即可登录
- 或用注册时的邮箱 + Magic Link 登录
- 两种方式任选一种
```

### 5.6.2 换手机号

商户后台有"修改账号手机号"功能：
1. 用旧手机号收一个验证码（确认身份）
2. 输入新手机号
3. 新手机号收验证码确认
4. 完成切换

## 5.7 注册数据收集

### 5.7.1 必填

- 手机号（账号 + 联系方式）

### 5.7.2 选填

- 邮箱（可选，用作 Magic Link 登录）
- 昵称（默认显示手机号后 4 位）

### 5.7.3 不收集（重要）

❌ **不要在注册时收集这些：**
- 真实姓名
- 身份证号
- 营业执照
- 银行账户
- 详细地址

理由：注册门槛越低越好。等用户用顺了再考虑增值服务。

## 5.8 不审核

第一版**不做注册审核**：
- 注册即用
- 用得人少时监控不过来也没必要
- 出问题再加机制（如举报、人工审核）

## 5.10 安全考虑

### 5.10.1 验证码

- 6 位数字
- 60 秒有效
- 同一手机号每分钟最多发 1 次
- 单日最多发 10 次（防刷）

### 5.10.2 登录 Token

- JWT 格式，30 天有效
- 服务器可主动失效（如检测异常）
- 用户可在"我的设备"页面查看所有登录设备
- 可远程登出某个设备（第二版功能）

### 5.10.3 隐私保护

- 手机号存储加密
- 不在前端展示完整手机号（如 `+60 12-3456****`）
- 提供"删除账户"功能
- 符合马来西亚 PDPA（个人资料保护法）
